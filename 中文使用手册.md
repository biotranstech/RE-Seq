RE-Seq: RNA编辑分析流程（中文使用指南）
=====
## 项目简介（Overview）
RE-Seq 是一个基于 Python DAGflow 构建的 RNA编辑（RNA editing）自动化分析流程，可以从原始 RNA-seq 数据（FASTQ）直接生成完整分析结果和可视化报告。

### RNA editing 介绍
RNA 编辑是一种动态的转录后修饰过程，在基因调控和疾病机制中具有重要意义。

### 核心功能
1. RNA-seq 质量控制
2. 比对到参考基因组
3. RNA 编辑检测
4. 多层次统计分析
5. 功能富集分析
6. 发表级可视化结果
7. 自动生成 HTML 报告

### 为什么要使用 RE-Seq（Why）
传统 RNA 编辑分析存在以下问题：
| 问题                    | 传统方式                                |
| ----------------------- | -------------------------------------- |
| 工具分散                 | ffastp + hisat2 + GATK + 多个脚本       |  
| 操作复杂                 | 多步骤手动执行                           |
| 易出错                   | 路径/格式/参数容易错                     |
| 难复现                   | 不同人结果不一致                         |

### RE-Seq 解决方案:
👉 一条命令完成全部流程：
✔ 自动质控
✔ 自动比对
✔ 自动 RNA 编辑检测
✔ 自动统计分析
✔ 自动绘图
✔ 自动生成 HTML 报告
👉 适用于：
1. RNA编辑研究
2. 转录组分析扩展
3. 生信服务项目
4. SCI论文数据分析

## 流程原理（Workflow）
```
FASTQ
 ↓
质量控制（fastp）
 ↓
比对（hisat2）
 ↓
BAM处理
 ↓
RNA编辑检测（GATK / REDItools 等）
 ↓
过滤与注释
 ↓
统计分析
 ↓
可视化
 ↓
HTML报告
```

## 项目结构
项目结构如下（核心模块说明）：
```
├── all.py                    # 主程序（入口）
├── dagflow                   # python DAGflow 包
├── report_noAS.html          # 报告模版
├── report_RNA_edic.py        # 自动化报告生成
├── report_utils.py           # 自动化报告分析模块
├── REseq                     # 核心模块
│   ├── common.py             # 公共文件处理模块
│   ├── config.py             # 软件路径配置（必须修改）
│   ├── down_analysis.py      # 下游分析
│   ├── gene_count.py         # 表达分析
│   ├── qc_hista.py           # 质控分析
│   └── work_bam_reseq.py     # RNA编辑检测
├── scripts                   # 分析脚本路径
├── template                  # 报告模版文件
│   ├── asset                 # ccs文件
│   └── images                # 报告插图
```

### 修改配置文件（必须）
编辑：
```
REseq/config.py
```
设置参考基因组路径：
```
ref_path = {"hg38": ".../Refer/hg38/hg38",
                        "mm10": ".../Refer/mm10/mm10",
                        "rn6":  ".../Refer/rn6/rn6"
}
ref_fasta = {"hg38": ".../Refer/hg38/hg38.fa",
                        "mm10": ".../Refer/mm10/mm10.fa",
                        "rn6":  ".../Refer/rn6/rn6.fa"
}
...
```
⚠️ 不修改这里 = 程序无法运行

### 输入文件准备（Input）
需要一个 sample.xls 文件：
```
SampleID   R1.fastq.gz   R2.fastq.gz   Group
S1         /path/1.fq.gz /path/2.fq.gz Tumor
S2         ...           ...           Tumor
S3         ...           ...           Normal
```
📌 格式说明
```
| 列    | 内容   |
| ---  | ---- |
| 第1列 | 样本名  |
| 第2列 | R1路径 |
| 第3列 | R2路径 |
| 第4列 | 分组信息 |
```

Usage
-----
### 查看帮助
您可以输入命令获取帮助信息，其中包含所有可选参数的信息:

```python
python all.py -h 
```

### 执行分析（核心命令）

```
python all.py \
  --work_dir ./work \
  --out_dir ./result \
  -ref hg38 \
  -i sample.xls \
  -cn PROJECT001 \
  -pn "RNA editing project"
```
⚙️ 常用参数说明
```
| 参数         | 说明                      |
| ---------- | ----------------------- |
| `-i`       | 输入样本表                   |
| `-ref`     | 参考基因组                   |
| `--thread` | 线程数                     |
| `-pn`       | 项目名称 |
```

### 常见问题
1. 软件路径错误
👉 必须检查：
```
REseq/config.py
```

2. 参考基因组未配置
需要：
```
fasta
index
```

3. 权限问题
```
chmod -R 755 项目目录
```

4. 环境冲突
```
conda clean -a
```
